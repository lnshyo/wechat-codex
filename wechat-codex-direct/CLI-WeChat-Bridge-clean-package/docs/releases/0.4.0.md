# CLI WeChat Bridge Release Notes - v0.4.0

## Overview

**v0.4.0** fixes critical reliability issues with the Codex panel connection and thread switching. This release ensures that the panel connection survives network hiccups, local input mirroring works correctly, and switching threads doesn't cause historical message replay in WeChat.

## Bug Fixes

### Codex Panel Connection Stability

- **Auto-Reconnection**: Panel now automatically reconnects when the websocket connection closes unexpectedly
- **Graceful Degradation**: Connection failures no longer terminate the entire bridge; reconnection is attempted first
- **Connection State Tracking**: Bridge now properly tracks connection state and retries failed RPC requests
- **Timeout Protection**: Reconnection attempts use configurable timeouts to prevent indefinite hangs

### Local Input Mirroring

- **Fixed Duplicate Messages**: Resolved issue where local input appeared twice in WeChat
- **Clean Mirror Source**: Local input now uses a single, reliable source (websocket structured events)
- **Session Log Fallback**: Added session log fallback when websocket is temporarily unavailable

### Thread Switch History Suppression

- **No More History Replay**: Switching threads no longer causes old messages to flood WeChat
- **Time-Based Gating**: Only messages newer than the switch timestamp are processed
- **Clean State Transitions**: Thread switches now properly reset session log parsing state

### Busy State Recovery

- **Ghost Active Turn Cleanup**: Fixed issue where "busy" state persisted after task completion
- **Proper Turn Finalization**: Active turns are now properly cleared when Codex finishes work
- **State Consistency**: Bridge state now correctly reflects idle/busy status at all times

## Important Notes

### Migration Notes

No breaking changes. This release is a drop-in replacement for v0.3.0.

### Behavioral Changes

- **Thread Switching Experience**: Thread switches are now faster and don't spam WeChat with historical messages
- **Connection Errors**: Most connection errors are now transient and auto-recovered
- **Busy State**: The bridge is more responsive to state changes; fewer false "still working" errors

## Technical Details

### Connection Resilience

- **RPC Reconnection**: Added `reconnectRpcClientAfterUnexpectedClose()` with exponential backoff
- **Request Recovery**: `sendRpcRequest()` now ensures connection before sending
- **Connection Lifecycle**: Properly track connection state through `isRpcSocketOpen()` and `ensureRpcClientConnected()`

### Thread Switch Logic

- **Session Ignore Window**: Added `sessionIgnoreBeforeMs` timestamp gating
- **Replay Filtering**: `shouldIgnoreCodexSessionReplayEntry()` filters historical messages
- **Switch Detection**: Thread changes trigger session reset and ignore window update

### State Management

- **Active Turn Lifecycle**: Proper cleanup in session `task_complete` handler
- **Busy State**: Fixed status transitions in websocket event handlers
- **Completion Detection**: Accurate turn completion detection from multiple sources
