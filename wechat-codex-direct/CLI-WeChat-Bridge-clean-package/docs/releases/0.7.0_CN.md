# CLI WeChat Bridge 发布说明 - v0.7.0

## 概述

**v0.7.0** 完成了 Claude 远程审批流程，并显著改进了微信审批消息格式。此版本确保 Claude Code 的权限请求正确转发到微信，并具有清晰、可操作的格式。

## 主要功能

### 完整的 Claude 远程审批

- **完整权限流程**：Claude Code 权限请求现在完全集成到微信审批系统
- **结构化审批**：权限请求被解析并格式化为清晰的微信消息
- **远程确认**：`/confirm` 和 `/deny` 命令通过 Claude hooks 工作
- **审批响应**：决策通过 hook 输出正确发送回 Claude Code

### 增强的审批格式

- **工具特定消息**：不同工具类型（Bash、ExitPlanMode、WebFetch 等）的不同消息格式
- **计划摘要**：长计划内容被智能摘要以供微信显示
- **清晰的行动项**：审批消息清楚显示正在请求什么操作
- **可读的详细信息**：命令路径、URL 和计划摘录被提取并格式化以提高可读性

## 改进

### Claude Hook 集成

- **Hook 脚本生成**：为 Windows 和 POSIX 环境动态创建脚本
- **权限负载**：正确解析 Claude 的权限请求结构
- **决策格式**：允许/拒绝决策的正确 hook 输出格式
- **错误处理**：优雅处理 hook 失败和超时

### 审批消息质量

- **不再有原始 JSON**：结构化审批数据转换为人类可读的消息（部分复杂情况可能还是要优化）
- **简洁摘要**：长计划被缩减为基本信息
- **工具名称提取**：清晰识别哪个工具正在请求权限
- **操作上下文**：提供足够的上下文以做出明智的审批决策

## 重要说明

### Claude 审批工作流程

当 Claude Code 请求权限时：

1. 桥接器通过 hook 接收权限请求
2. 请求被解析并格式化以供微信使用
3. 用户在微信中看到清晰的审批消息
4. 用户使用 `/confirm` 或 `/deny` 响应
5. 决策通过 hook 输出发送回 Claude Code
6. Claude 根据决策继续执行

### 消息格式示例

**Bash 命令**：
```
[需要审批]
工具：Bash
命令：npm install -g package-name
操作：批准或拒绝此命令执行
```

**计划模式**：
```
[需要审批]
工具：ExitPlanMode
计划：实现版本检查功能...
摘要：添加自动更新检测和通知
操作：批准或拒绝计划完成
```

## 技术细节

### 新组件

- **`claude-hooks.ts`**：完整的 hook 中继和权限处理实现
- **Hook 脚本模板**：特定于平台的 hook 脚本生成
- **权限解析器**：提取和格式化权限请求详细信息
- **决策格式化器**：将用户决策转换为 hook 输出格式

### 审批流程

```
Claude Code → Hook → 桥接器 → 微信
                     ↓
                   用户
                     ↓
微信 → 桥接器 → Hook → Claude Code
```

### 消息格式化

- **工具识别**：从权限请求中识别工具类型
- **详细信息提取**：提取相关的命令/路径/URL/计划信息
- **摘要化**：将长内容浓缩为基本信息
- **模板应用**：应用适当的消息模板
