# CLI WeChat Bridge Release Notes - v0.2.0

## Overview

**v0.2.0** represents a complete redesign of the Codex WeChat bridge, introducing a dual-terminal architecture that preserves the native Codex TUI experience while providing clean, structured final replies through WeChat. This release fundamentally changes how the bridge works, moving away from embedding Codex in a pseudo-terminal to running a native Codex panel alongside a structured websocket-based remote control interface.

This is the first stable release of the bridge functionality and establishes the foundation for all future development.

## Major Features

### Native Codex Terminal Experience

- **Native TUI Panel**: Codex now runs in a true native terminal with full TUI support, not an embedded PTY simulation
- **Visible Panel**: Users see the real Codex interface with all original features intact—alt-screen, cursor navigation, and interactive controls work exactly as expected
- **No Text Injection Degradation**: The terminal is driven directly by Codex itself, eliminating character-by-character injection artifacts and display corruption

### Structured Remote Control via WeChat

- **Clean Final Replies**: WeChat receives only the final answer text, not terminal fragments, TUI redraws, or internal state noise
- **Structured Turn Management**: Remote messages are sent through Codex's app-server websocket API, not PTY keystroke injection
- **Task Completion Detection**: The bridge accurately identifies when Codex finishes a task and sends a single, coherent reply

- **No Protocol Pollution**: Intermediate signals like `task_complete`, duration, heartbeat messages, and status bar text are filtered from the WeChat chat stream

### Simplified Authorization Model

- **Removed `/pair` Mechanism**: Eliminated the temporary pairing workflow in favor of direct credential binding
- **Single Owner Model**: The WeChat account in `account.json.userId` is the only authorized user
- **No Runtime Rebinding**: Owner identity is established once during setup and never changes during runtime

### Improved Windows Compatibility

- **PowerShell Script Avoidance**: Automatically detects and bypasses `codex.ps1` to avoid PowerShell execution policy issues
- **Vendor exe Priority**: Prefers the official `codex.exe` from vendor directories over staging copies
- **Robust Path Resolution**: Handles both `.cmd` and `.exe` entry points with correct command-line escaping

## Architecture Changes

### Dual-Process Design

The bridge now consists of two coordinated processes:

1. **Bridge Process** (`wechat-bridge-codex`): Manages We connectivity and maintains structured turn state
2. **Panel Process** (`wechat-codex`): Runs the native Codex TUI with full terminal capabilities

This separation enables:
- True native terminal behavior
- Structured remote control without PTY hacks
- Clean separation of concerns and easier debugging

### Codex App-Server Integration

- Direct websocket connection to Codex's app-server for structured operations
- Support for `thread/start`, `turn/start`, `turn/interrupt` RPC calls
- Server-side approval handling for command execution and file changes
- Proper `experimentalApi` capability negotiation

## Important Notes

### Breaking Changes from Pre-Releases

If you were using experimental versions before v0.2.0:

- **No More `/pair` Command**: Delete any saved pairing state; authorization now uses `account.json` directly
- **Panel Entry Point Changed**: Use `wechat-codex` instead of embedded modes
- **Thread State Migration**: Shared threads are now persisted and restored across bridge restarts
- **PTY Artifacts Eliminated**: Any workflows that depended on parsing PTY output will need adjustment

### Migration Actions

1. Ensure your WeChat credentials are properly saved in `~/.claude/channels/wechat/account.json`
2. Restart the bridge and panel to establish the new architecture
3. Verify that your account's `userId` is recognized as the authorized owner
4. Test native Codex panel interaction—keyboard input and display should work exactly like standalone Codex

## Upgrade Instructions

From versions prior to v0.2.0:

```bash
# Pull the latest code
cd CLI-WeChat-Bridge
git pull

# Install dependencies
bun install

# Reinstall global commands
npm install -g .

# Start the bridge in your project directory
cd D:\work\your-project
wechat-bridge-codex

# In a second terminal, start the Codex panel
wechat-codex
```
