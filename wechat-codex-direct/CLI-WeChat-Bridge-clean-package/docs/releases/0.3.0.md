# CLI WeChat Bridge Release Notes - v0.3.0

## Overview

**v0.3.0** introduces workspace-isolated state management and the ability to run global bridge commands from any directory. This release transforms the bridge from a single-workspace tool into a true global CLI utility while maintaining clean state separation between different projects.

## Major Features

### Global Commands with Workspace Isolation

- **Run from Any Directory**: Bridge commands now work globally from any directory, not just the project repository
- **Automatic Workspace Detection**: The current working directory automatically becomes the active workspace
- **Isolated State per Workspace**: Each project maintains its own bridge state, preventing cross-project interference
- **Global Binary Installation**: Install once with `npm install -g .` and use `wechat-bridge-codex` anywhere

### Workspace State Management

- **Workspace-Specific State Files**: State is stored in `~/.claude/channels/wechat/workspaces/<workspace-key>/`
- **Stable Workspace Keys**: Generated from normalized absolute paths using SHA256 hash
- **Isolated Panel Endpoints**: Each workspace has its own panel endpoint file
- **Clean Workspace Separation**: Multiple projects can run bridges simultaneously without state conflicts

### Simplified Installation

- **Single Global Installation**: `npm install -g .` makes commands available everywhere
- **Development Mode**: Use `npm link` for development without reinstalling
- **Force Update Option**: `npm install -g . --force` to refresh the installation

## Architecture Changes

### Data Directory Structure

```
~/.claude/channels/wechat/
├── account.json                 # Global WeChat credentials
├── bridge.lock.json             # Global runtime lock (single bridge)
└── workspaces/
    └── <workspace-key>/         # Per-workspace data
        ├── bridge-state.json    # Workspace-specific state
        └── codex-panel-endpoint.json  # Workspace panel endpoint
```

### Runtime Lock Strategy

- **Single Bridge Instance**: Global lock prevents multiple bridges from consuming the same WeChat message stream
- **Per-Workspace State**: While only one bridge runs at a time, each workspace maintains independent state
- **Clean Recovery**: Bridge crashes or restarts don't affect other workspaces' persisted data

## Important Notes

### Breaking Changes

If you were previously running the bridge from within the repository:

1. **Reinstall Global Commands**: Run `npm install -g .` to register the global commands
2. **Update Launch Workflow**: Start the bridge from your actual project directory, not the repository
3. **Workspace Migration**: Existing bridge state is automatically migrated to workspace-isolated files

### New Workflow

**Before (repository-bound)**:
```bash
cd CLI-WeChat-Bridge
bun run bridge:codex
```

**After (global commands)**:
```bash
cd D:\work\your-project
wechat-bridge-codex
```

### Panel Command Updates

- **Old**: `bun run codex:panel`
- **New**: `wechat-codex-panel` (global command)

## Upgrade Instructions

```bash
# Navigate to the repository
cd CLI-WeChat-Bridge

# Pull the latest code
git pull

# Install dependencies
bun install

# Install global commands
npm install -g .

# Now you can run from any project directory
cd D:\work\your-project
wechat-bridge-codex

# In a second terminal (same directory)
wechat-codex-panel
```
