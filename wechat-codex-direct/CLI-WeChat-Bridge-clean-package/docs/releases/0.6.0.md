# CLI WeChat Bridge Release Notes - v0.6.0

## Overview

**v0.6.0** is a quick fix release that restores missing imports after the bridge adapter refactoring in v0.5.0. This release ensures that all bridge adapters can properly access shared utility functions.

## Bug Fixes

### Import Restoration

- **Missing Helper Functions**: Restored imports for shared bridge utilities that were lost during directory reorganization
- **Adapter Compatibility**: Fixed imports in `bridge-adapters.codex.ts`, `bridge-adapters.claude.ts`, and `bridge-adapters.shell.ts`
- **Runtime Errors**: Resolved `is not defined` errors that occurred when bridge adapters attempted to use shared functions

## Affected Components

All three bridge adapters were affected:
- Codex adapter
- Claude adapter
- Shell adapter

## Important Notes

### What Was Fixed

The v0.5.0 release reorganized the bridge adapter code into separate files but accidentally removed import statements for shared utilities. This release restores those imports.

### No Breaking Changes

This is a bug-fix release with no API or behavioral changes.

## Technical Details

### Files Modified

- **`src/bridge/bridge-adapters.codex.ts`**: Restored imports from `bridge-adapters.shared.ts`
- **`src/bridge/bridge-adapters.claude.ts`**: Restored imports from `bridge-adapters.shared.ts`
- **`src/bridge/bridge-adapters.shell.ts`**: Restored imports from `bridge-adapters.shared.ts`

### Impact

- **Before**: Bridge adapters would crash at runtime with `ReferenceError: [function] is not defined`
- **After**: All shared utilities are properly accessible to all adapters
