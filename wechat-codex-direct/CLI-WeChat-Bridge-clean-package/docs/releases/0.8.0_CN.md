## 1. 版本检查和自动更新功能

### 背景

此前，用户必须手动下载源代码来更新微信桥接器。此版本引入了基于网络的版本检查机制，类似于 npm 的更新工作流，为将来的 npm 发布做准备，同时保持当前的安装方法。

### 实现

**基于 Git 的版本发现**
- 使用 `git ls-remote --tags origin` 获取可用版本
- 避免了困扰早期实现的 GitHub API 速率限制
- 直接从远程仓库解析语义版本标签
- 高效的单命令执行，开销最小

**智能缓存策略**
- 24小时缓存，存储在 `~/.claude/channels/wechat/update-check.json`
- 防止冗余网络请求
- 启动后 3 秒运行非阻塞检查
- 静默失败，不影响桥接器运行

**版本比较逻辑**
- 语义版本比较（major.minor.patch）
- 准确检测新版本
- 区分相等、大于和小于版本

**手动检查命令**
```bash
wechat-check-update
```

提供专业输出，包括：
- 当前版本
- 最新可用版本
- 清晰的更新说明
- 无表情符号，专业呈现

## 2. 媒体发送和附件解析

### 背景

此前，微信桥接器只能发送文本消息。用户无法通过桥接器发送图片、文件、语音消息或视频。此版本集成了全面的媒体发送功能，将独立的 `wechat-notify.ts` 脚本合并到主传输层。

### 实现

**媒体类型支持**
- **图片**：PNG、JPG、JPEG、GIF、WebP、BMP（最大 20MB）
- **文件**：任何文件类型（最大 50MB）
- **语音**：MP3、WAV、M4A、OGG、AAC（最大 20MB）
- **视频**：MP4、MOV、MKV、AVI、WebM（最大 100MB）

**附件路径解析**
桥接器现在可以智能解析多种附件路径格式：

1. **围栏代码块**
   `````markdown
   ```wechat-attachments
   image C:\Users\user\Desktop\screenshot.png
   file /path/to/document.pdf
   video ~/Videos/recording.mp4
   voice C:\Music\voice_note.m4a
   ```
   ``````

2. **内联路径**
   - Windows 绝对路径：`C:\Users\user\file.png`
   - 家目录相对路径：`~/Desktop/file.png`
   - maas-log URL：`https://server.com/C:\path\to\file.png`

3. **自动类型检测**
   - 基于文件扩展名推断
   - 支持 Windows 和 Unix 路径
   - 处理 URL 编码路径

**双队列架构**
防止大文件上传阻塞文本消息：

- **文本队列**：高优先级队列，立即发送文本
- **附件队列**：独立的媒体上传队列
- 两个队列独立运行
- 失败的附件不会阻止后续消息

**最终回复转发模块**
新的 `bridge-final-reply.ts` 模块：
- 分离可见文本和附件
- 立即发送文本
- 按顺序上传和发送附件
- 每个附件的优雅错误处理
- 报告失败的附件而不崩溃

**媒体大小限制**
通过环境变量配置：

```bash
export WECHAT_MAX_IMAGE_MB=20   # 默认：20MB
export WECHAT_MAX_FILE_MB=50    # 默认：50MB
export WECHAT_MAX_VOICE_MB=20   # 默认：20MB
export WECHAT_MAX_VIDEO_MB=100  # 默认：100MB
```

**Codex 集成**
- Codex 最终输出统一为 `final_reply` 事件
- 文本和附件正确分离
- 自动附件检测和发送

**MCP 工具暴露**
更新 MCP 工具定义以暴露：
- `wechat_send_image`
- `wechat_send_file`
- `wechat_send_voice`
- `wechat_send_video`

## 3. 微信长轮询失败预防

### 背景

此前，微信长轮询期间的任何网络错误都会导致整个桥接器崩溃，需要手动重启。这在网络连接不稳定的环境中尤其成问题。此版本引入了全面的错误分类和重试逻辑。

### 实现

**错误分类**
传输错误现在分为四类：

1. **超时错误**
   - 请求超时
   - 连接超时
   - 头部超时
   - 可用指数退避重试

2. **网络错误**
   - 连接被拒绝
   - 连接重置
   - 主机不可达
   - 网络不可达
   - 可用指数退避重试

3. **HTTP 错误**
   - HTTP 状态码：408、425、429（可重试）
   - 其他 4xx/5xx 代码（可能不可重试）
   - 上下文相关处理

4. **认证错误**
   - 认证失败
   - 不可重试（需要用户干预）
   - 立即退出

**指数退避重试**
- 基础延迟：1 秒
- 最大延迟：30 秒
- 指数增长：2^consecutiveFailures
- 限制在第 5 次幂（32 秒）

**详细错误日志**
新的错误描述系统包括：
- 错误名称和消息
- 错误代码（errno）
- 系统调用（syscall）
- 主机名和地址
- 端口信息
- 原因链（最多 4 级深度）

**不可恢复错误处理**
- 认证失败仍然立即退出
- 防止无限重试循环
- 清晰的错误消息供用户操作
