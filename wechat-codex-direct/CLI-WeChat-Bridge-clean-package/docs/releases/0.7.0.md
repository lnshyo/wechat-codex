# CLI WeChat Bridge Release Notes - v0.7.0

## Overview

**v0.7.0** completes the Claude remote approval flow and significantly improves WeChat approval message formatting. This release ensures that Claude Code's permission requests are properly forwarded to WeChat with clear, actionable formatting.

## Major Features

### Complete Claude Remote Approval

- **Full Permission Flow**: Claude Code permission requests now fully integrate with WeChat approval system
- **Structured Approvals**: Permission requests are parsed and formatted into clear WeChat messages
- **Remote Confirmation**: `/confirm` and `/deny` commands work through Claude hooks
- **Approval Response**: Decisions are properly sent back to Claude Code via hook output

### Enhanced Approval Formatting

- **Tool-Specific Messages**: Different message formats for different tool types (Bash, ExitPlanMode, WebFetch, etc.)
- **Plan Summarization**: Long plan content is intelligently summarized for WeChat display
- **Clear Action Items**: Approval messages clearly show what action is being requested
- **Readable Details**: Command paths, URLs, and plan excerpts are extracted and formatted for readability

## Improvements

### Claude Hook Integration

- **Hook Script Generation**: Dynamic script creation for Windows and POSIX environments
- **Permission Payloads**: Proper parsing of Claude's permission request structures
- **Decision Formatting**: Correct hook output format for allow/deny decisions
- **Error Handling**: Graceful handling of hook failures and timeouts

### Approval Message Quality

- **No More Raw JSON**: Structured approval data is converted to human-readable messages
- **Concise Summaries**: Long plans are truncated to essential information
- **Tool Name Extraction**: Clear identification of which tool is requesting permission
- **Action Context**: Sufficient context to make informed approval decisions

## Important Notes

### Claude Approval Workflow

When Claude Code requests permission:

1. Bridge receives permission request via hook
2. Request is parsed and formatted for WeChat
3. User sees clear approval message in WeChat
4. User responds with `/confirm` or `/deny`
5. Decision is sent back to Claude Code via hook
6. Claude continues execution based on decision

### Message Format Examples

**Bash Command**:
```
[Approval Required]
Tool: Bash
Command: npm install -g package-name
Action: Approve or deny this command execution
```

**Plan Mode**:
```
[Approval Required]
Tool: ExitPlanMode
Plan: Implement version checking feature...
Summary: Add automatic update detection and notifications
Action: Approve or deny plan completion
```

## Technical Details

### New Components

- **`claude-hooks.ts`**: Complete hook relay and permission handling implementation
- **Hook Script Templates**: Platform-specific hook script generation
- **Permission Parsers**: Extract and format permission request details
- **Decision Formatters**: Convert user decisions to hook output format

### Approval Flow

```
Claude Code → Hook → Bridge → WeChat
                     ↓
                   User
                     ↓
WeChat → Bridge → Hook → Claude Code
```

### Message Formatting

- **Tool Recognition**: Identifies tool type from permission request
- **Detail Extraction**: Pulls out relevant command/path/URL/plan info
- **Summarization**: Condenses long content to essentials
- **Template Application**: Applies appropriate message template
