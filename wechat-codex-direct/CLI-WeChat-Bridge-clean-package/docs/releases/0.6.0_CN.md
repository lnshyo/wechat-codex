# CLI WeChat Bridge 发布说明 - v0.6.0

## 概述

**v0.6.0** 是一个快速修复版本，恢复了 v0.5.0 中桥接适配器重构后丢失的导入。此版本确保所有桥接适配器可以正确访问共享实用函数。

## 错误修复

### 导入恢复

- **缺少的辅助函数**：恢复目录重组期间丢失的共享桥接实用工具的导入
- **适配器兼容性**：修复了 `bridge-adapters.codex.ts`、`bridge-adapters.claude.ts` 和 `bridge-adapters.shell.ts` 中的导入
- **运行时错误**：解决了桥接适配器尝试使用共享函数时出现的 `is not defined` 错误

## 受影响的组件

所有三个桥接适配器都受到影响：
- Codex 适配器
- Claude 适配器
- Shell 适配器

## 重要说明

### 修复内容

v0.5.0 版本将桥接适配器代码重新组织为单独的文件，但意外删除了共享实用工具的导入语句。此版本恢复了这些导入。

### 无重大变更

这是一个错误修复版本，没有 API 或行为变更。

## 升级说明

```bash
# 导航到仓库
cd CLI-WeChat-Bridge

# 拉取最新代码
git pull

# 安装依赖
bun install

# 重新安装全局命令
npm install -g .
```

## 技术细节

### 修改的文件

- **`src/bridge/bridge-adapters.codex.ts`**：恢复从 `bridge-adapters.shared.ts` 的导入
- **`src/bridge/bridge-adapters.claude.ts`**：恢复从 `bridge-adapters.shared.ts` 的导入
- **`src/bridge/bridge-adapters.shell.ts`**：恢复从 `bridge-adapters.shared.ts` 的导入

### 影响

- **之前**：桥接适配器在运行时崩溃，显示 `ReferenceError: [function] is not defined`
- **之后**：所有共享实用工具都可以被所有适配器正确访问
