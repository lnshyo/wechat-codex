# CLI WeChat Bridge 发布说明 - v0.4.0

## 概述

**v0.4.0** 修复了 Codex 面板连接和线程切换的关键可靠性问题。此版本确保面板连接在网络波动中存活，本地输入镜像正常工作，切换线程不会在微信中导致历史消息重放。

## 错误修复

### Codex 面板连接稳定性

- **自动重新连接**：当 websocket 连接意外关闭时，面板现在自动重新连接
- **优雅降级**：连接失败不再终止整个桥接器；首先尝试重新连接
- **连接状态跟踪**：桥接器现在正确跟踪连接状态并重试失败的 RPC 请求
- **超时保护**：重新连接尝试使用可配置的超时以防止无限挂起

### 本地输入镜像

- **修复重复消息**：解决了本地输入在微信中出现两次的问题
- **清晰的镜像源**：本地输入现在使用单一、可靠的来源（websocket 结构化事件）
- **会话日志回退**：当 websocket 暂时不可用时添加了会话日志回退

### 线程切换历史抑制

- **不再有历史重放**：切换线程不再导致旧消息在微信中泛滥
- **基于时间的门控**：仅处理比切换时间戳更新的消息
- **清晰的状态转换**：线程切换现在正确重置会话日志解析状态

### 忙碌状态恢复

- **幽灵活动轮次清理**：修复了任务完成后"忙碌"状态持续存在的问题
- **正确的轮次完成**：当 Codex 完成工作时，活动轮次现在被正确清除
- **状态一致性**：桥接状态现在始终正确反映空闲/忙碌状态

## 重要说明

### 迁移说明

无重大变更。此版本是 v0.3.0 的直接替代品。

### 行为变更

- **线程切换体验**：线程切换现在更快，不会用历史消息轰炸微信
- **连接错误**：大多数连接错误现在是瞬时的并自动恢复
- **忙碌状态**：桥接器对状态变化响应更快；"仍在工作"错误更少

## 技术细节

### 连接弹性

- **RPC 重新连接**：添加了 `reconnectRpcClientAfterUnexpectedClose()`，具有指数退避
- **请求恢复**：`sendRpcRequest()` 现在在发送之前确保连接
- **连接生命周期**：通过 `isRpcSocketOpen()` 和 `ensureRpcClientConnected()` 正确跟踪连接状态

### 线程切换逻辑

- **会话忽略窗口**：添加了 `sessionIgnoreBeforeMs` 时间戳门控
- **重放过滤**：`shouldIgnoreCodexSessionReplayEntry()` 过滤历史消息
- **切换检测**：线程更改触发会话重置和忽略窗口更新

### 状态管理

- **活动轮次生命周期**：在会话 `task_complete` 处理程序中正确清理
- **忙碌状态**：修复了 websocket 事件处理程序中的状态转换
- **完成检测**：从多个来源准确检测轮次完成

## 文件统计

- **修改的文件**：3 个核心桥接文件
- **更改的行数**：约 200 行插入，约 50 行删除
- **测试覆盖**：添加了连接恢复和线程切换测试
