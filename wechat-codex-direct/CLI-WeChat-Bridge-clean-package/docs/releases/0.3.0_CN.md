# CLI WeChat Bridge 发布说明 - v0.3.0

## 概述

**v0.3.0** 引入了工作区隔离的状态管理和从任何目录运行全局桥接命令的能力。此版本将桥接器从单工作区工具转变为真正的全局 CLI 实用程序，同时在不同的项目之间保持清晰的状态分离。

## 主要功能

### 具有工作区隔离的全局命令

- **从任何目录运行**：桥接命令现在可以从任何目录全局运行，而不仅仅是项目仓库
- **自动工作区检测**：当前工作目录自动成为活动工作区
- **每个工作区隔离状态**：每个项目维护自己的桥接状态，防止跨项目干扰
- **全局二进制安装**：使用 `npm install -g .` 安装一次，随处使用 `wechat-bridge-codex`

### 工作区状态管理

- **工作区特定状态文件**：状态存储在 `~/.claude/channels/wechat/workspaces/<workspace-key>/`
- **稳定的工作区密钥**：使用 SHA256 哈希从规范化绝对路径生成
- **隔离的面板端点**：每个工作区都有自己的面板端点文件
- **清晰的工作区分离**：多个项目可以同时运行桥接器而不会发生状态冲突

### 简化的安装

- **单次全局安装**：`npm install -g .` 使命令在任何地方都可用
- **开发模式**：使用 `npm link` 进行开发而无需重新安装
- **强制更新选项**：`npm install -g . --force` 刷新安装

## 架构变更

### 数据目录结构

```
~/.claude/channels/wechat/
├── account.json                 # 全局微信凭据
├── bridge.lock.json             # 全局运行时锁（单桥接器）
└── workspaces/
    └── <workspace-key>/         # 每工作区数据
        ├── bridge-state.json    # 工作区特定状态
        └── codex-panel-endpoint.json  # 工作区面板端点
```

### 运行时锁策略

- **单桥接器实例**：全局锁防止多个桥接器消耗同一微信消息流
- **每工作区状态**：虽然一次只运行一个桥接器，但每个工作区维护独立状态
- **清晰恢复**：桥接器崩溃或重启不影响其他工作区的持久数据

## 重要说明

### 重大变更

如果您之前从仓库内运行桥接器：

1. **重新安装全局命令**：运行 `npm install -g .` 以注册全局命令
2. **更新启动工作流程**：从实际项目目录启动桥接器，而不是仓库
3. **工作区迁移**：现有桥接状态自动迁移到工作区隔离文件

### 新工作流程

**之前（仓库绑定）**：
```bash
cd CLI-WeChat-Bridge
bun run bridge:codex
```

**之后（全局命令）**：
```bash
cd D:\work\your-project
wechat-bridge-codex
```

### 面板命令更新

- **旧**：`bun run codex:panel`
- **新**：`wechat-codex-panel`（全局命令）

## 技术细节

### 新文件

- **`channel-config.ts`**（新增）：工作区路径规范化和密钥生成
- **`codex-panel.ts`**（增强）：添加了 `--cwd` 参数以指定工作区
- **`codex-panel-link.ts`**（增强）：工作区感知的端点文件 I/O

### 配置变更

- **`package.json`**：添加了带有全局命令映射的 `bin` 字段
- **`bin/*.mjs`**：所有全局命令的新包装器脚本
